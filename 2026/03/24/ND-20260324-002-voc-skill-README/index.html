VoC (Voice of Customer) 技能

📚 简介

VoC（Voice of Customer）技能用于记录、分析和响应用户（客户）反馈，将用户的声音转化为可执行的决策和文档更新。

核心功能

  1. 如实记录用户发言

    • 记录时间戳
    • 记录发言内容
    • 记录会话上下文

  2. 发掘有价值信息

    • 分析用户需求
    • 识别潜在问题
    • 发现改进机会
    • 更新相关文档

🎯 VoC技能使用指南

触发场景

VoC技能会在以下情况自动激活：

  1. 用户反馈意见：

    • “我觉得…”
    • “我希望…”
    • “建议…”

  2. 用户提出需求：

    • “需要…”
    • “想要…”
    • “能否…”

  3. 用户遇到问题：

    • “问题…”
    • “错误…”
    • “不行…”
    • “失败…”

  4. 用户表达偏好：

    • “喜欢…”
    • “不喜欢…”
    • “习惯…”

不触发场景

  • 简单询问：如”今天天气怎么样？”
  • 命令执行：如”重启服务”、”发布博客”
  • 临时对话：无实质内容的寒暄

🏷️ 分类体系

分类维度

VoC技能使用5大分类维度进行分析：

1. NPS情感分类（基于Net Promoter Score理论）

分类 评分范围 特征 应对策略
推广者 9-10分 极其满意，推荐给他人 🔔 请求推荐、分享成功案例
被动者 7-8分 满意但缺乏热情 🔄 了解原因、提升体验、转化为推广者
贬损者 0-6分 不满意，可能批评 ⚠️ 优先解决问题、深入理解痛点、重建信任

2. 反馈类型分类（基于实际产品反馈）

分类 子分类 优先级 示例
功能需求 新功能 🔴 高 “希望添加xxx功能”
功能改进 🟡 中 “xxx功能可以优化”
功能删除 🟢 低 “xxx功能不需要”
问题报告 Bug 🔴 高 “xxx报错了”
性能问题 🔴 高 “xxx很慢”
兼容性问题 🟡 中 “xxx不兼容”
体验反馈 易用性 🟡 中 “xxx不好用”
界面设计 🟡 中 “xxx界面很乱”
交互流程 🟡 中 “xxx流程太繁琐”
偏好表达 喜好 🟢 低 “我喜欢xxx”
不喜欢 🟢 低 “我不喜欢xxx”
习惯 🟢 低 “我习惯xxx”
改进建议 优化建议 🟡 中 “建议xxx改进”
创新想法 🟡 中 “可以考虑xxx”

3. 业务影响分类（基于对业务的影响）

分类 影响维度 价值评分 示例
战略影响 影响长期方向、核心竞争力 9-10分 “影响核心竞争力”
产品影响 影响产品功能、性能、质量 7-8分 “影响主要功能”
用户体验影响 影响用户满意度、使用习惯 5-6分 “影响用户体验”
运营影响 影响工作效率、流程 3-4分 “影响工作效率”
无影响 一般性建议、闲聊 1-2分 “一般性建议”

4. 数据源分类（基于反馈来源）

数据源 可靠性 分析方法
直接反馈 ⭐⭐⭐⭐ 直接分析
间接反馈 ⭐⭐⭐ 需要推断
隐性需求 ⭐⭐ 数据分析

5. 生命周期分类（基于产品发展阶段）

分类 说明 处理方式
发现阶段 新用户首次接触 记录、观察、引导
采纳阶段 用户开始使用 支持答疑、收集反馈
保留阶段 用户持续使用 深化价值、提升体验
推荐阶段 用户推荐他人 收集成功案例、请求推荐

💡 价值评分系统

评分计算

总分 = 重要性 + 可行性 + 影响范围

维度 评分范围 说明
重要性 1-3分 战略级(3) / 核心功能(3) / 次要功能(2) / 一般建议(1)
可行性 1-3分 容易实现(3) / 需要设计(2) / 需要重构(1) / 暂不可行(0)
影响范围 1-4分 全局影响(4) / 模块影响(2) / 局部影响(1)

评分示例

| 场景 | 重要性 | 可行性 | 影响范围 | 总分 | 评级 |
|——|——–|——–|———-|——|
| 战略级需求 + 易实现 + 全局影响 | 3 | 3 | 4 | 10 | 🏆 极高价值 |
| 核心功能 + 需要设计 + 模块影响 | 3 | 2 | 2 | 7 | ⭐ 高价值 |
| 次要功能 + 容易实现 + 局部影响 | 2 | 3 | 1 | 6 | 🌟 中等价值 |
| 一般建议 + 需要重构 + 局部影响 | 1 | 1 | 1 | 3 | 💡 低价值 |
| 信息性查询 | 1+1+1 | 3 | ⭐⭐ 低价值/信息性 |

评级标准

  • 9-10分：🏆 极高价值 → 优先处理
  • 7-8分：⭐ 高价值 → 近期规划
  • 5-6分：🌟 中等价值 → 中期规划
  • 3-4分：💡 低价值 → 暂存档
  • 1-2分：📝 信息性 → 仅记录

🚀 后台处理策略

自动记录模式

VoC技能会在后台静默处理所有用户发言，只在必要时主动通知用户。

通知条件

条件 说明
用户主动询问 用户明确要求查看分析结果
发现关键问题 影响核心功能或安全的问题
发现高价值需求 价值评分 ≥ 9分的重大需求
定期汇总 每天或每周汇总时推送

静默处理

  • 日常对话：如实记录到VoC日志
  • 常规分析：在后台进行分类和价值评分
  • 汇总报告：定期生成VoC汇总报告

减少打扰

用户希望的日常对话正常进行，不必要的分析结果推送不会打扰到聊天。

📋 标签系统

标签使用规则

  1. 主标签优先：每个记录至少1个主标签（反馈类型）
  2. 次标签补充：根据情况添加情感标签、影响标签
  3. 标签数量控制：每条记录最多3-5个标签
  4. 标签一致性：使用标准化标签，避免同义词混用

常用标签

分类 标签示例
反馈类型 需求, 功能, 新功能, 功能改进
问题报告 问题, bug, 错误, 失败
体验反馈 体验, 易用性, 界面设计
偏好表达 偏好, 喜好, 不喜欢, 习惯
改进建议 建议, 优化, 改进
NPS情感 推广者, 满意, 不满, 批评
业务影响 战略, 产品, 用户体验, 运营

🔄 工作流程

记录阶段

  1. 接收用户消息
  2. 生成记录条目（时间戳、发送者、内容）
  3. 初始分类（分类决策树）
  4. 计算价值评分
  5. 添加标签
  6. 写入日志文件

分析阶段

  1. 读取日志文件
  2. 统计分析（按日期、用户、标签、分类）
  3. 识别趋势和模式
  4. 发现高价值信息

更新阶段

根据分析结果，更新以下文档：

  1. MEMORY.md：添加新的用户偏好
  2. 规则文件：更新USER.md、SOUL.md
  3. 任务记录：创建任务记录，跟踪需求实现进度
  4. VoC汇总：定期生成VoC汇总报告

📊 汇总报告

报告类型

  1. 月度汇总：每月生成一次
  2. 季度汇总：每季度生成一次
  3. 高价值报告：针对高价值信息生成专项报告

报告内容

  • 统计数据（总记录数、有价值信息数量）
  • 价值评分Top 10
  • 分类统计
  • 主要发现和趋势
  • 待处理项

🎓 最佳实践

对于用户

  1. 清晰表达：明确表达你的需求、问题和偏好
  2. 提供上下文：必要时提供更多背景信息
  3. 及时反馈：对VoC分析和响应给予反馈

对于技能维护者

  1. 定期维护：定期查看日志和汇总报告
  2. 更新分类体系：根据业务发展调整分类维度和标签
  3. 优化价值评分：根据实际效果调整评分标准
  4. 确保数据安全：VoC日志包含用户信息，需要妥善保护
