B站视频自动解读：三级降级的内容提取流水线

本文最后更新于 2026年6月5日 晚上

需求

自动提取B站视频的语音内容，生成文本摘要。

核心矛盾：B站视频有字幕的不到一半，大部分视频只能靠音频转写。所以需要一条能自动降级的流水线——有字幕就拿字幕，没有就下载音频做 Whisper 转写，最后兜底拿元信息。

架构

硬件分工：音频下载在本地完成，Whisper 推理走远程 GPU 服务器（通过 SSH + SCP 联动）。

GPU 0 跑着 ComfyUI，Whisper 必须用 GPU 1。medium 模型只需约 400MB 显存，GPU 1 的 18GB 空间绰绰有余。

搜索能力

除了从 URL 提取内容，还需要搜索B站视频。B站搜索 API 同样有 412 反爬问题。

解法： 通过本地部署的 SearXNG 搜索引擎，用 site:bilibili.com 限定搜索范围，再从结果中提取 BV 号，最后调B站视频信息 API 获取元数据（标题、播放量、时长等）。

总结

这套方案的核心思路：

  1. 三级降级——优先快路径，失败自动降级，永远有兜底
  2. API 直连代替第三方工具——yt-dlp 被封就用 B站自己的 playurl API
  3. 硬件分工——CPU 活在本地干，GPU 活通过 SSH 远程调用
  4. 模型选择——medium 而不是 large-v3，400MB 显存换可接受的中文识别率

整个流水线从零到跑通大约一个下午，大部分时间花在踩坑上。但每个坑都有解，而且解法都比较稳定——毕竟用的都是官方 API 和成熟工具，不是 hack。
